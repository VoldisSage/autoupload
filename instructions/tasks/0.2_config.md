## Description
Implement a configuration system based on a JSON file located in `config/config.json`. Provide a TypeScript module that loads and validates configuration values.

## Definition of Done
- A file `config/config.json` exists with example keys, including:
  - `twitch` (clientId, clientSecret, broadcasterId, pollIntervalSeconds)
  - `obs` (recordingsPath)
  - `openai` (apiKey placeholder, model, promptVersion)
  - `youtube` (OAuth client information)
  - `discord` (webhookUrl)
- A module `src/config.ts` exists and:
  - Loads `config/config.json` synchronously at startup.
  - Validates that required sections and keys are present.
  - Throws a clear error (with meaningful message) if config is missing or invalid.
  - Exports a typed `config` object for use in other modules.
- The application exits with a non-zero status if configuration is not valid.

## Tests
- Run `npm run start` with a valid `config/config.json` and confirm it starts correctly.
- Temporarily rename `config/config.json` and run `npm run start`:
  - Expect: process exits and prints a clear configuration error.
- Set a required field (e.g. `twitch.clientId`) to an empty string and run `npm run start`:
  - Expect: process exits and prints a clear configuration error.

## Files to create / modify
- `config/config.json` (example values, NOT real secrets)
- `src/config.ts`
- Update `README.md` with a short section about configuration.
