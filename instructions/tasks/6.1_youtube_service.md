## Description
Implement a YouTube upload service that can either perform a real upload using the YouTube Data API or simulate the upload when `dryRun` is enabled.

## Definition of Done
- A file `src/services/youtubeService.ts` exists with:
  - `async function uploadVideoToYoutube(filePath: string, metadata: YoutubeMetadata, options?: { dryRun?: boolean }): Promise<string>`
- When `options?.dryRun === true`:
  - The function does NOT call the real YouTube API.
  - It logs a message indicating a dry run.
  - It returns a fake URL, e.g. `https://www.youtube.com/watch?v=TEST_VIDEO_ID`.
- When `dryRun` is not set or is `false`:
  - The function uses `googleapis` and OAuth2 credentials from `config.youtube`.
  - It uploads the video file at `filePath` to YouTube with:
    - `snippet.title` from `metadata.title`
    - `snippet.description` from `metadata.description`
    - `snippet.tags` from `metadata.tags`
    - `snippet.categoryId` set to `"20"` (Gaming)
    - `status.privacyStatus` defaulting to `"unlisted"`
  - On success, returns the real YouTube watch URL: `https://www.youtube.com/watch?v=<videoId>`.
- Errors are caught and rethrown with meaningful `Error` messages.

## Tests
- Create `dev/testYoutubeDryRun.ts` that:
  - Calls `uploadVideoToYoutube("C:/ruta/fake.mp4", fakeMetadata, { dryRun: true })`.
  - Logs the returned URL.
- Add npm script:
  - `"dev:test-youtube-dry": "ts-node dev/testYoutubeDryRun.ts"` or equivalent.
- Run `npm run dev:test-youtube-dry` and verify:
  - The function completes successfully and returns a fake URL.
- (Optional, manual) Later, once OAuth2 is configured:
  - Implement a separate test for a real upload and manually verify the video appears in your YouTube channel.
