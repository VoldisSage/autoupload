## Description
Implement a Twitch service that can fetch an app access token and query whether the configured broadcaster is currently live.

## Definition of Done
- A file `src/services/twitchService.ts` exists with:
  - `async function getAppAccessToken(): Promise<string>`
  - `async function isStreamLive(): Promise<boolean>`
- `getAppAccessToken`:
  - Uses `twitch.clientId` and `twitch.clientSecret` from `config`.
  - Calls Twitch OAuth endpoint to obtain an app access token.
  - Caches the token in memory until it expires (basic caching is enough).
- `isStreamLive`:
  - Uses the cached token (or fetches a new one if needed).
  - Calls `https://api.twitch.tv/helix/streams?user_id=<broadcasterId>`.
  - Returns `true` if there is at least one stream object, `false` otherwise.
- Errors are caught and rethrown with meaningful messages.

## Tests
- Create `dev/testTwitchLive.ts` that:
  - Calls `isStreamLive()` once and logs `Live: true` or `Live: false`.
- Add npm script:
  - `"dev:test-twitch": "ts-node dev/testTwitchLive.ts"` or equivalent.
- Run `npm run dev:test-twitch` with the Twitch channel offline:
  - Expect output: `Live: false` (or similar).
- (Optional manual) Run again with the channel live and verify `Live: true`.
