## Description
Implement functionality to create a human-readable Markdown file for each processed video, containing metadata and context about the stream.

## Definition of Done
- `src/services/metadataStore.ts` defines a type `StreamContext` with at least:
  - `videoId: string`
  - `twitchStreamId: string`
  - `youtubeVideoId?: string`
  - `dateISO: string`
  - `durationSeconds: number`
  - `channelName: string`
  - `twitchTitle: string`
  - `startedAt: string`
  - `endedAt: string`
  - `modelName: string`
  - `promptVersion: string`
- A function `saveMarkdown(metadata: YoutubeMetadata, ctx: StreamContext): Promise<void>` (or sync) exists that:
  - Ensures `data/videos/` directory exists.
  - Builds a filename: `YYYY-MM-DD_<slug>.md` using date + channel + videoId or game.
  - Writes a Markdown file with:
    - A frontmatter-like header (key: value lines) including IDs, dates, game, language, model, etc.
    - A section for title, description, tags, chapters, and context details.
- The resulting `.md` file is human-readable and easy to inspect.

## Tests
- Create `dev/testMetadataMd.ts` that:
  - Creates a fake `YoutubeMetadata` and `StreamContext`.
  - Calls `saveMarkdown`.
  - Logs the path of the created file.
- Add npm script:
  - `"dev:test-md": "ts-node dev/testMetadataMd.ts"` or equivalent.
- Run `npm run dev:test-md` and verify:
  - A `.md` file is created in `data/videos/`.
  - Opening the file shows expected fields and sections.
